# Medical Terminology Agent

# Introduction

The Medical Terminology Agent is a simple AI application that explains medical terms in easy-to-understand language. It is built using Python and Amazon Bedrock (Nova Micro) and is deployed on Amazon Bedrock AgentCore Runtime.

The application is intended only for educational purposes. It does not diagnose medical conditions or provide treatment recommendations.


# Technologies Used

- Python
- Amazon Bedrock
- Amazon Nova Micro
- Amazon Bedrock AgentCore Runtime
- Amazon S3
- GitHub Actions
- Pytest

# How it Works

1. The user enters a medical term.
2. The input is validated.
3. The application sends the request to Amazon Bedrock.
4. Bedrock generates a simple explanation.
5. If Bedrock is unavailable, the application retrieves the explanation from the local knowledge base.
6. The response is returned to the user.


# Running the Application

Install the required packages.

```bash
pip install -r requirements.txt
```

Run the application locally.

```bash
python run.py
```


# Running with AgentCore Runtime

```bash
python main.py
```

The runtime starts locally on port **8080** and accepts requests through the AgentCore endpoint.


# Running Tests

Execute all unit tests.

```bash
pytest
```

Generate the coverage report.

```bash
pytest --cov=medical_terminology_agent --cov-report=term-missing
```


# Deployment

The project includes a GitHub Actions workflow.

Whenever code is pushed to the **main** branch, the workflow:

- Installs project dependencies
- Executes unit tests
- Creates the deployment package
- Uploads the deployment package to Amazon S3

The uploaded package can then be used to create or update the Amazon Bedrock AgentCore Runtime.
