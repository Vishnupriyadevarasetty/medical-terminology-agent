# Execution Steps
This document describes the steps I followed to build, test, and deploy the Medical Terminology Agent.

# Local Setup
First, I cloned the repository and created a Python virtual environment. After activating the environment, I installed all the required packages using the requirements file.
Command used:

```bash
pip install -r requirements.txt
```

The application uses Amazon Bedrock, so I configured my AWS credentials and selected the required AWS Region and Bedrock model.


# Running the Application
For local testing, I used the following command:

```bash
python run.py
```

The application prompts for a medical term and returns a simple explanation generated by Amazon Bedrock.


# Running Unit Tests
After completing the implementation, I verified the functionality by running the unit tests.

```bash
pytest
```

All unit tests passed successfully.


# Checking Code Coverage
The project requirement was to achieve at least 90% code coverage.

I generated the coverage report using:

```bash
pytest --cov=medical_terminology_agent --cov-report=term-missing
```

The project achieved **100% code coverage**.


# Running AgentCore Runtime
Before testing the deployed runtime, I verified the application entry point.

```bash
python main.py
```

After deployment, I also tested the runtime using the AgentCore CLI to confirm that it was returning the expected responses.


# CI/CD Pipeline
After the project was pushed to GitHub, the GitHub Actions workflow automatically performed the following tasks:

- Installed project dependencies
- Executed unit tests
- Verified code coverage
- Created the deployment package
- Uploaded the deployment package to Amazon S3

This ensured that every change pushed to the **main** branch generated an updated deployment package.

# AgentCore Deployment
After the deployment package was uploaded to Amazon S3, I deployed the application to Amazon Bedrock AgentCore Runtime.

The runtime uses:

- Deployment package from Amazon S3
- Entry point: `main.py`
- AWS IAM execution role

After deployment, I verified that the runtime status changed to **Ready** and tested it using the AgentCore CLI to confirm that it was responding correctly.
