# Execution Steps
This document describes the steps I followed to build,test and deploy the Medical Terminology Agent.

# Local Setup
First,I cloned the repository and created a Python virtual environment. After activating the environment, I installed all the required packages using the requirements file.

Command used:
pip install -r requirements.txt

The project uses Amazon Bedrock, so I created a '.env' file to store the required configuration values.

AWS_REGION=us-east-1
BEDROCK_MODEL_ID=amazon.nova-micro-v1:0

# Running the Application
For local testing, I used the command below.

python run.py

The application prompts for a medical term and returns a simple explanation generated by Amazon Bedrock.

# Running Unit Tests
Once the implementation was complete,I verified the functionality by running the unit tests.

pytest

All tests passed successfully.

# Checking Code Coverage
The project requirement was to achieve at least 90% code coverage.
I generated the coverage report using:

pytest --cov=medical_terminology_agent --cov-report=term-missing

The final coverage achieved.

# Running AgentCore Runtime Locally
Before deploying to AWS, I verified that the AgentCore Runtime worked locally.

python main.py

After starting the runtime,I tested the endpoint using a POST request to confirm that the runtime was returning the expected response.

# CI/CD Pipeline
After the project was pushed to GitHub,a GitHub Actions workflow automatically performed the following tasks:
-Installed project dependencies
-Executed unit tests
-Verified code coverage
-Created a deployment package
-Uploaded the package to Amazon S3

This ensured that every change pushed to the main branch generated an updated deployment package.

# AgentCore Deployment
Once the deployment package was available in Amazon S3, I created an Amazon Bedrock AgentCore Runtime.
The runtime uses:
-Deployment package from Amazon S3
-Entry point:'main.py'
-Default IAM role created by AWS

After deployment,the runtime status changed to 'Ready' confirming that the application was successfully deployed.
